# jz examples — perf vs V8 (Node, jz optimize:speed)

Generated by the perf-opportunity-map workflow. jzVsV8 = V8_ms / jz_ms (>1 = jz faster). f64x2 = SIMD ops emitted.

## Delivered — per-pixel-color vectorizer (`tryPerPixelColor`, commit 577dd4f)

A dual of the escape-time vectorizer for pixel loops with NO inner loop: compute an f64 value from
the index (cos/sin/sqrt…), pack to a u32 colour, store. Two adjacent pixels → f64x2 lanes
(index→ramp, transcendentals→bit-exact `$math.*2` mirrors, conditionals→`v128.bitselect`), then the
scalar pack+store per lane; the original loop is the odd-pixel tail. **Bit-exact to scalar**
(verified across odd/even widths). Measured best-of-3, 800×500×60 frames:

| example | before | after vs V8 | vs scalar jz | how |
|---|---|---|---|---|
| **chladni** | 1.86× | **2.28×** | 1.43× | Phase 1: two `Math.cos` → `$math.cos2` |
| **interference** | ~1.0× | **1.14×** | 1.26× | Phase 1 sin+sqrt 2-wide + Phase 2 `a**γ` → `$math.pow2` |

Deterministic pins (machine-independent, run by `npm test`): `test/simd.js` — bit-exact SIMD-vs-no-
vectorize + path ratchets + odd-width tail; `test/examples.js` — the real chladni/interference files.

The other "per-pixel-iter" candidates below are NOT blocked by missing transcendentals (the Phase-2
premise): voronoi/metaballs/lyapunov are per-pixel inner-loop REDUCTIONS, domain-color's sqrt is in
an inlined-hypot control-flow block, bifurcation/plasma are array-load / f64-field maps — each a
separate, larger vectorizer. raymarcher is correctly declined (its `march` writes a global → an
impure call the pass can't reorder bit-exactly).

The table below is the original baseline snapshot (chladni/interference rows updated to ✓).
